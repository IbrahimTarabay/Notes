-->Bugs cost companies a lot of money

-->Any serious company is going to have tests

TDD-->test driven development, you actually write test before write your application code.
--------------------------------------------------------------

Unit Tests-->test individual functions or classes

Integration Tests-->This is testing how different pieces of code work together so it can be testing of whether the database works with express app that's requesting some data

Automation Tests-->UI tests, This usually involves testing real life scenarios on the browser by making sure that the expected behavior on the web is correct, it's simulation of user behavior in browser like clicking,typing,scrolling
