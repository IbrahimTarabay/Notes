COMMANDS SYNTAX

Commands typically have the syntax:
  command-options(s)-arguments(s)
ex: #ls -l bart (command-option-argument)

Options:
1-Modify the way that a command works
2-Usually consist of a hyphen or dash followed by a single letter(-l)
3-Some commands accept multiple options which can usually be grouped together after a single hyphen (-tlr)

Arguments:
1-Most commands are used together with one or more arguments
2-Some commands assume a default arguments if none is supplied
3-Arguments are optional for some commands and required by others

FILE PERMISSIONS

-->UNIX is a multi-user system. Every file and directory in your account can be protected from or made accessible to other users by changing its access permissions. Every user has responsibility for controlling access to their files

-->Permissions for a file or directory may be restricted to by types

-->There are 3 type of permissions(r-read, w-write, x-execute)

-->Each permission (rwx) can be controlled at three levels:
 u user=yourself
 g group=can be people in the same project
 o other=everyone on the system

-->File or Directory permission can be displayed by running ls -l command
 rwxrwxrwx

-->Command to change permission
 chmod

ex:
chmod g-w jerry (mean delete write permission from group in the file jerry, notice (command-option-argument))

ex:
chmod a-r jerry (mean delete read permission form user,group,other in the file jerry)
----------------------------------------------------------------------------
FILE OWNERSHIP

-->There are 2 owners of a file or directory
  -User and Group(which user belongs)

-->Command to change file ownership
 chown-->changes the ownership of a file
 chgrp-->changes the group ownershop of a file

-->Recursive ownership change option (Cascade)
 #-R
(it will not only change that directory but also all the files and directories within that parent directory)

ex:
#chown root lisa (mean change owner from ibrahim to root on file lisa)
ex:
#chgrp root lisa (mean change group from ibrahim to root on file lisa)

Standard Output to a File (tee)

-->"tee" command is used to store and view (both at the same time) the output of any command



ex:
#echo "David Puddy is Eliane's boy friend" | tee eliane
#echo "David and Eliane" | tee -a eliane
(-a to add new line without delete the previous)
(eliane is the file)

ex:
#wc -c eliane (to know how many characters eliane file has)
