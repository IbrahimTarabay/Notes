What is MVC?

1-MVC is a programming design pattern or methodology

2-It is an architectural pattern separating an application into three main logical components: the model, the view, and controller

3-Its separates your application presentation from your business logic

4-Each part are designed to perform specific task


Benefits of MVC

1-Reusability

2-Easy maintain our application

3-Many developers work at a time without affecting one another

4-it supports TTD (Test-Driven Development)

Eloquent vs Query Builder(DB Query)

Eloguent better in:

1-To create a better relationship and get the results in view with so much simple syntax, when there needs to join

2-Eloquent is also for who have not much knowledge of SQL query

3-An MVC framework follow the rules of code readability, code maintainability and which eloquent is

ex:
// In Eloquent 
$student = App\Student::find($id);
// In DB facade
$student = DB::table('student')->where('id', $id)->first();

So when we use Eloquent and when DB facades:

1-When we'll work on a simple and small records site with simple CRUD and there records are not fact, then use Eloquent there.

2-When we'll work on a lot's of records, it's better to us DB Query than eloquent

ex:

1-University website which may contain maxium 5000 teachers and 10000 students and some notices and files then it's better to do this with simple Laravel Eloquent which is very much standard and readable

2-Stackoverflow which may contains more than 10000000(posts and many more things) conventional DB facades there, it's more faster for searching the posts from so much records
