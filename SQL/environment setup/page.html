Setting Up Windows for command line

-->if we want to use command line as our main source of interacting with PostgreSQL

1-open start menu

2-click on edit system environment variables

3-click on Environment Variables

4-go to System variables and click on path(the path variable, is the variable that is going to be loaded by CMD and subsequently power shell and any other command prompt that you may install or want to use)

path variable-->in essence will point to all the binaries, all the executables(the software basically that allows you to run certain commands)

5-click edit on path variable(the most important is C:\Program Files\PostgreSQL\13\bin)

bin-->stands for binaries and it adds commands so we can use (the most important one is psql)

-->When we install software, basically what's we're doing is we're installing "Binaries" so when we add the bin folder(C:\Program Files\PostgreSQL\13\bin) to the path variable we say hey command prompt, go and actually use this folder and load all of these binaries for me so that i can run these commands inside of the system so that i don't have to reference them directly from where they at , you're going to know about them forever

6-open PowerShell and write psql -U postgres
