Third-party code

-->Using existing third-party libraries or packages in your code

for example:
-->library for sending email, or writing to a log file or a template engine etc

Downloading code libraries

-->Download packages and save them with the rest of your code

Dependencies
-->Packages dependent on other packages so you need to download and install those as well, doing all of this can take a lot of time, fortunately there is a tool that do all of that to you called composer

Composer-->Dependency Manager for PHP

So how does it work?

1-You specify which libraries you need in your project and you do this in a file called composer.json-->it's just a text file which lists package names and versions, you then run the composer program which read this file and then downloads and installs the packages into a folder in your project
--------------------------------------------------------
Checking composer by writing this command

$composer about
-->to know if you have composer

$composer install
-->This will download and save all the packages and their dependencies into your project automatically

$composer require "twig/twig:~1.0"
-->to install one package

composer.lock-->This contains a list of the exact versions of each of the packages installed into the project, This file is generated and updated automatically, so you never need to open it and edit it manually

packagist-->is the website where you can find packages
--------------------------------------------------------
Include all package classes automatically using the composer autoloader


-->when you use a class that's defined in a separate file, you have to require that file first


-->you can use an autoloader to load these classes automatically so you don't have to require each one individually


-->Many third-party packages provide their own autoloader so if a package has many classes and you want to use these classes, all you need to do is to include the package's autoloader


-->So for each different third-party package that you're using in a project, you need to find and require its autoloader


-->if you're using composer to install the packages you don't need to do this as composer includes its own autoloader that automatically includes all the autoloaders from each individual package so by simply including composer's autoloader in your code you get autoloading of all the classes in all of your packages with just one require statement


---------------------------------------------------------
Updating the Composer autoloader

$composer dump-autoload
