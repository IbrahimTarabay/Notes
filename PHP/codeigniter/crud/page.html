-->Codeigniter has control of filters which lets us run code before or after every method in a controller

Filters-->are classes that implement the codeignitor filter interface and that contain a before and after method that run code before and after all the controller methods respectively
--------------------------------------------------------
-->Codeignitor comes with CSRF protection built in, it runs as a filter, so the first step in activating it is to enable it in the filter configuration

CSRF-->works by generating a random token, storing this in the session and comparing this to a value supplied in post requests

Pagination

-->to show tasks a group at time, codeigniter has simple but flexible pagination library that will do this for us, This is loaded automatically with the model, so we don't need to explicitly load it in the tasks controller

-->to view the rest of the records, we need pagination links so we can navigate from one page to another, to do this first, we need to get the instance of the page library that the model is using, this is set as a property on the model automatically
